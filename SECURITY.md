# Security Policy

## Supported Versions

The following table lists the versions of the Django APIs App that are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |

We recommend that all users and organizations use the latest supported versions to ensure they receive the latest security updates and patches.

### Reporting a Vulnerability
- We take the security of our project very seriously. If you believe you have found a security vulnerability in the Django APIs App, we encourage you to let us know right away. We will investigate all legitimate reports and do our best to quickly fix the problem.

### How to Report a Vulnerability
- Email Us Directly: Send an email to our security team at mshariq28022000@gmail.com with the details of the vulnerability. Please include as much information as possible to help us understand and reproduce the issue.

### Provide a Detailed Description: 
When reporting a vulnerability, please provide a clear and concise description of the issue, including:
- The type of vulnerability (e.g., XSS, SQL Injection, etc.)
- The affected version(s) of the Django APIs App
- Steps to reproduce the vulnerability
- Any additional details or proof of concept (PoC) scripts.
- Encrypt Sensitive Information: If your report contains sensitive information, we encourage you to encrypt your email using our public PGP key, which is available here.

### What to Expect
- Acknowledgment: We will acknowledge receipt of your vulnerability report within 48 hours.
- Initial Assessment: Our security team will review the report and determine the severity of the vulnerability. This initial assessment will be communicated to you within 72 hours of the report's receipt.
- Resolution Timeline: For valid security vulnerabilities, we aim to release a fix within 7 to 14 days, depending on the complexity and severity of the issue. In some cases, it may take longer to address complex vulnerabilities.
- Notification: You will be notified when the vulnerability has been patched and a security release is available.
- Credit: We appreciate the efforts of security researchers and are happy to acknowledge your contributions. If you prefer to remain anonymous, please let us know.


### Disclosure Policy
We ask that you do not publicly disclose the vulnerability until we have had a chance to investigate and address it. We are committed to transparency and will publicly disclose the details of the vulnerability after a fix has been released and users have been given adequate time to update their installations.

### Security Best Practices
Always use the latest stable version of the Django APIs App.
Regularly update all dependencies to their latest secure versions.
Follow our Best Practices Guide for securing your deployment.
